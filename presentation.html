Task 1

LiteLLM Proxy — Claude CLI as an API

Bridges the claude CLI into an OpenAI-compatible endpoint. Any client that speaks OpenAI can use Claude — no SDK changes.

OpenAI request
→
LiteLLM CustomLLM
→
claude -p
Key implementation details
  • ClaudeCLIProvider(CustomLLM) in custom_handler.py
  • acompletion() + astreaming() async methods
  • Multi-turn history → Human: …\nAssistant: …
  • Tools serialised into --system-prompt, JSON parsed back
  • _SUBPROCESS_ENV = dict(os.environ) — avoids uvloop rejection
  • 120-second timeout for Claude CLI subprocess
Demo — curl 
# Non-streaming
curl -X POST https://litellm-proxy-vici.zeabur.app/v1/chat/completions \
  -H "Authorization: Bearer $KEY" # $KEY = LiteLLM master key (set in Zeabur dashboard) \
  -H "Content-Type: application/json" \
  -d '{
    "model": "claude-cli/claude-sonnet-4-5",
    "messages": [{"role":"user","content":"Hello"}]
  }'

# Streaming — returns SSE chunks
curl ... -d '{..., "stream": true}'

# Tool use
curl ... -d '{
  ...,
  "tools": [{
    "type": "function",
    "function": {
      "name": "get_weather",
      "description": "Get current weather",
      "parameters": { ... }
    }
  }]

Task 2

GitHub Skills — CI/CD as Claude Commands

Four .md skill files in github-skills/skills/, symlinked into .claude/commands/. Invoke directly from Claude Code — no copy-paste, no context switching.

/gh-status
List recent workflow runs. Read-only — safe to call any time.
repo (required) · limit (default: 10)
/gh-test
Trigger test.yml, poll until complete, return pass/fail + URL.
repo (required) · branch (default: main)
/gh-lint
Trigger lint.yml, poll with 10 s intervals (max 5 min).
repo (required) · branch (default: main)
/gh-deploy
Deploy with mandatory CONFIRM gate. Auto-detects repo & changed service.
repo · branch · environment · service
Safety design
  • Deploy requires user to type CONFIRM before dispatch
  • Validator warns on embedded credentials / Bearer tokens
  • validator.py enforces schema: name, description, version, inputs, outputs
  • test_all_skills_valid() runs in CI — all 4 files must pass
Auto-detect in /gh-deploy 
# Detect repo from git remote
git remote get-url origin
# → https://github.com/user/repo.git

# Detect changed service from diff
git diff --name-only HEAD~1
# litellm-proxy/** → service=litellm-proxy
# browser-agent/** → service=browser-agent
# both / neither   → service=all

# Symlink install
ln -s ../github-skills/skills .claude/commands
Task 3

Browser Agent — Natural Language → Web Actions

Tool-use loop
U
POST /run-task
{"task": "What is the latest Python release?"}
C
Claude decides: call browser_search
tool_calls → {name: "browser_search", args: {…}}
T
Execute: DuckDuckGo search
Result added back as tool message
C
Claude decides: navigate + extract_text
Reads python.org/downloads body (≤8 000 chars)
✓
Claude calls done(answer)
Loop exits — returns answer + full logs
6 browser tools
  • browser_search(query) — DuckDuckGo HTML (no API key)
  • navigate(url) — Playwright goto, returns title
  • extract_text() — page body ≤ 8 000 chars
  • click(selector) — Playwright click
  • type_text(selector, text) — Playwright fill
  • done(answer) — terminates loop with result
